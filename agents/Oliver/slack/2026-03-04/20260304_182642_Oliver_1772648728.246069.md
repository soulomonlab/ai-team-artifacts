## Database Design for 세계 시총 Top 10 주식 분석 + 전세계 증시 뉴스 요약 + 가격 변동 알림 봇

목표 요약
- 전세계 시가총액 Top N(기본 10) 주식의 시세/시총 이력 수집 및 분석
- 전세계 증시/경제 뉴스 수집, 요약, 감성 분석 저장
- 사용자별 가격 변동 알림(임계치/비율/키워드 기반) 관리 및 전송
- 실시간/근실시간 조회용 API와 백그라운드 수집/처리 워크플로우 지원

### 설계 고려사항(요약)
- 대상 종목은 Top 10 위주지만 확장 가능하게 설계
- 시세는 고빈도(예: 1m)로 수집 → 파티셔닝(또는 TimescaleDB hypertable)을 권장
- 뉴스는 원문(raw JSON) 저장 + 요약/감성 메타데이터 추가
- 알림은 idempotent하게 처리, 재전송/스냅샷 보존
- 실시간 알림 발행엔 Redis/Kafka 같은 메시지 큐 사용 권장

### Tables
- stocks
  - id BIGSERIAL PRIMARY KEY
  - symbol VARCHAR(32) NOT NULL -- e.g., AAPL, 005930.KS
  - name VARCHAR(255)
  - primary_exchange_id INT REFERENCES exchanges(id)
  - country_code CHAR(2)
  - isin VARCHAR(20)
  - cusip VARCHAR(20)
  - sector VARCHAR(100)
  - industry VARCHAR(200)
  - latest_market_cap NUMERIC(26,2) -- 최신 시총
  - latest_market_cap_at TIMESTAMP WITH TIME ZONE
  - active BOOLEAN DEFAULT TRUE
  - created_at TIMESTAMP WITH TIME ZONE DEFAULT now()
  - updated_at TIMESTAMP WITH TIME ZONE DEFAULT now()
  - UNIQUE(symbol)

- exchanges
  - id SERIAL PRIMARY KEY
  - code VARCHAR(20) UNIQUE
  - name VARCHAR(255)
  - timezone VARCHAR(64)

- market_cap_snapshots
  - id BIGSERIAL PRIMARY KEY
  - stock_id BIGINT REFERENCES stocks(id) ON DELETE CASCADE
  - market_cap NUMERIC(26,2) NOT NULL
  - rank INT -- global rank at snapshot time
  - snapshot_at TIMESTAMP WITH TIME ZONE NOT NULL -- when snapshot taken
  - source VARCHAR(100) -- 데이터 소스
  - UNIQUE (stock_id, snapshot_at)

- price_points
  - id BIGSERIAL PRIMARY KEY
  - stock_id BIGINT REFERENCES stocks(id) ON DELETE CASCADE
  - exchange_id INT REFERENCES exchanges(id)
  - price NUMERIC(20,6) NOT NULL
  - volume BIGINT -- 거래량(가능하면 tick/1m)
  - open_price NUMERIC(20,6)
  - high_price NUMERIC(20,6)
  - low_price NUMERIC(20,6)
  - ts TIMESTAMP WITH TIME ZONE
