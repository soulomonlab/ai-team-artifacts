Write-File Enforcement Policy

이 문서는 모든 산출물(코드, 스펙, 테스트, 문서)이 write_file 도구를 통해 생성되어야 한다는 규칙과 절차를 정의합니다.

핵심 요약
- 모든 산출물은 output/ 하위 디렉토리에 write_file을 사용해 생성한다.
- Slack/PR/이슈에서 output/* 경로를 언급할 때는 해당 파일이 실제로 존재해야 하며, 작성자는 write_file 호출 기록을 남긴다.
- QA는 변경사항을 검증하고 준수율을 보고한다.

절차
1. 산출물 작성자는 write_file으로 파일을 생성한다.
2. QA는 샘플링 또는 자동화된 검사로 output/에 파일이 존재하는지 검증한다.
3. 불일치 발견 시 PR/Slack에서 참조된 경로와 실제 파일 생성 기록을 요청한다.

수용 기준
- 새로운 PR에서 참조된 모든 output/* 파일 경로는 실제 파일 존재를 만족해야 함.
- QA 초기 목표 준수율 90%.

책임
- 작성자: write_file로 파일 생성
- QA: 검증 및 보고
- PO: 정책 유지 및 강화
